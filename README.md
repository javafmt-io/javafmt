# javafmt

A fast, opinionated Java formatter for Java 21+. No options to configure, no XML files to export, no whitespace debates in code review.

## Prerequisites

JDK and Gradle versions are pinned in `.sdkmanrc` via [SDKMAN!](https://sdkman.io/). There's no Gradle wrapper checked in — SDKMAN is doing that job.

```bash
# One-time SDKMAN install — see https://sdkman.io/install
curl -s "https://get.sdkman.io" | bash

# From the repo root:
sdk env install
sdk env
```

Set `sdkman_auto_env=true` in `~/.sdkman/etc/config` and `sdk env` runs every time you `cd` in.

## Build

```bash
gradle build
```

Everything goes through Gradle, including the VS Code extension's TypeScript compile. Don't reach for `npm` or `mvn` directly.

## Modules

| Module                 | What it does                                                                  |
|------------------------|-------------------------------------------------------------------------------|
| `javafmt-core`         | The parser and formatter. The only runtime dep is `slf4j-api`.                |
| `javafmt-cli`          | Standalone CLI. Ships as a fat JAR plus GraalVM native binaries per OS.       |
| `javafmt-spotless`     | A `FormatterStep` so any Gradle project on Spotless can pick it up.           |
| `javafmt-maven-plugin` | Maven plugin (`javafmt:format`, `javafmt:check`). Built by Gradle.            |
| `javafmt-intellij`     | IntelliJ plugin — wires into `Reformat Code` via `ExternalFormatProcessor`.   |
| `javafmt-vscode`       | VS Code extension. Talks NDJSON to a bundled native daemon (jar fallback).    |

The Maven plugin's JAR and `plugin.xml` descriptor are generated by Gradle via `org.gradlex.maven-plugin-development` — no `mvn install` step to bootstrap.
